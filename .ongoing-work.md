# Ongoing Work Log (Local, Ignored)

Purpose: keep interruption-safe notes about current unfinished execution work.

## How to use

- Update this file before/after significant changes.
- Keep entries short and action-oriented.
- Add exact commands used for validation.
- Record blockers and next concrete step.
- Mark each task with status: `PARTIAL`, `TODO`, or `BLOCKED` (completed work goes to `changelog.md`).
- Group and prioritize unfinished work.
- n8n is for development orchestration only (not product runtime behavior). when creating workflow it should be tested fully that it reports executions and works end to end error in work flow goes to P0 priority!
- Any n8n workflows created for dev cycles must be listed in this file with Name, ID, and Purpose.
- Use `pnpm run ongoing:bridge` outputs as the interface for feeding ongoing-work objectives into n8n automation.

## Autonomous Execution Rules

Agent may continue automatically without confirmation when:

- Task is a UI migration
- Task is refactor only
- No database schema changes
- No API contract changes
- No destructive file deletions

If these conditions are met:
→ Do NOT ask for confirmation
→ Continue execution until task acceptance criteria are met.

Human confirmation is required only for:

- at moment there is no situation where you should need human intervention
